Variables and Types

Variables in python are assigned by the equal sign '='
For example: x=1, y='abc' z = True
Python automatically assumes type, meaning in these cases, it knows x is an int, y is a string and z is a boolean.

{% endcall %} {% call section('numbers') %}

Numbers

Now, what do those types mean? These are default python types that python can operate on; others include Float, Decimal... The int type refers to whole numbers. Ints have the operators +(addition) -(subtraction) *(multiplication) and /(division) an example:

{% call macros.python_code() %} >>> 2 + 2 4 >>> 50 - 5*6 20 >>> (50 - 5*6) / 4 5.0 >>> 8 / 5 # division always returns a floating point number 1.6 {%- endcall %}

The integer numbers (2,4,20) have type int, the ones with a decimal (5.0,1.6) have type float. Division always returns a float
To get floor division use the operator // and for remainder use the operator %

{% call macros.python_code() %} >>> width = 20 >>> height = 5 * 9 >>> width * height 900 {% endcall %}

See how variables are used? Variables can be set to almost any name excluding some python operators like 'for' or 'in'.

Strings

Python can also manipulate strings. Strings are words or sets of characters surrounded by "" or '' Here's an example:

{% call macros.python_code() %} >>> 'spam eggs' # single quotes 'spam eggs' >>> 'doesn\'t' # use \' to escape the single quote... "doesn't" >>> "doesn't" # ...or use double quotes instead "doesn't" >>> '"Yes," he said.' '"Yes," he said.' >>> "\"Yes,\" he said." '"Yes," he said.' >>> '"Isn\'t," she said.' '"Isn\'t," she said.' {% endcall %}

Strings can be concatenated together with the + operator and repeated with *

{% call macros.python_code() %} >>> 'hi' + ' ' + 'there' hi there >>> 2*'yo' yoyo {% endcall %}

Since strings can be looked at as a list of characters, they can be indexed, with the first character having an index of 0. In python there is no type for character, they are just strings with a length of one.

{% call macros.python_code() %} >>> word = 'Python' >>> word[0] # character in position 0 'P' >>> word[5] # character in position 5 'n' {% endcall %}

Indices may also be negative numbers, if you would like to start counting from the right:

{% call macros.python_code() %} >>> word[-1] # last character 'n' >>> word[-2] # second-last character 'o' >>> word[-6] 'P' {% endcall %}

Attempting to use an index that is too large will result in an error:

{% call macros.python_code() %} >>> word[42] # the word only has 6 characters Traceback (most recent call last): File "stdin", line 1, in module IndexError: string index out of range {% endcall %} {% endcall %}
