Variables and Types

Variables in Python are assigned using the assignment operator =. The assignment operator is the same character as the equals sign. {% call utils.code('python') %} >>> x = 1 >>> y = 'abc' >>> z = True {% endcall %} Python automatically understands each variable's type, meaning that in the above example, it knows x is of type int, y is of type str, and z is of type bool.

{% endcall %} {% call section('numbers', 'Numbers') %}

Numbers

Now, what do those types mean? These are default types that Python can operate on. Other default types include float, Decimal, list, dict, and more.

The int type refers to whole numbers. int supports the following arithmetic operations:

  • Addition (+)
  • Subtraction (-)
  • Multiplication (*)
  • Division (/)
  • Integer division (//)
  • Modulo (%)
  • Exponentiation (**)
Here's an example:

{% call utils.code('python') %} >>> 2 + 2 4 >>> 50 - 5 * 6 # Python understands the order of operations... 20 >>> (50 - 5 * 6) / 4 # ...but you can also use parentheses 5.0 >>> 8 / 5 # Exact division; always returns a float 1.6 >>> 5 ** 2 # 5 raised to the power 2 25 {% endcall %}

The numbers 2, 4, and 20 have type int, whereas 5.0 and 1.6 have type float.

Division always returns a float. To perform floor division (divide and round down), use the integer division operator (//). To find the remainder of a division, use the modulo operator (%).

{% call utils.code('python') %} >>> width = 20 >>> height = 5 * 9 >>> width * height 900 {% endcall %}

Now do you see how variables are used? Variables can be set to almost any name, excluding some keywords reserved by Python that have special meanings (e.g., for, in, and not).

Strings

Python can also manipulate strings. A string (denoted by the type str) is a sequence of characters surrounded by matching sets of quotation marks.

Here's an example:

{% call utils.code('python') %} >>> 'spam eggs' # single quotes 'spam eggs' >>> 'doesn\'t' # Use \' to "escape", or ignore the single quote... "doesn't" >>> "doesn't" # ...or use double quotes instead "doesn't" >>> '"Yes," he said.' '"Yes," he said.' >>> "\"Yes,\" he said." '"Yes," he said.' >>> '"Isn\'t," she said.' '"Isn\'t," she said.' >>> """Some long ... line""" # You can also use triple quotes for multi-line strings 'Some long\nline' # \n is a special character that creates a new line {% endcall %}

Strings can be concatenated together with the addition (+) operator, and repeated with the multiplication (*) operator:

{% call utils.code('python') %} >>> 'hi' + ' ' + 'there' hi there >>> 2 * 'yo' yoyo {% endcall %}

Since strings act like a list of characters, they can be indexed. The first character has the index 0 (this language property is known as "zero-indexing", and many languages share this property). In Python, characters don't have their own type. They are simply strings of length 1.

{% call utils.code('python') %} >>> word = 'Python' >>> word[0] # Gets the character in position 0 'P' >>> word[5] # Gets the character in position 5 'n' {% endcall %}

If you want to index starting from the end of the string, you can pass a negative index instead. Note that reverse indexing starts at -1 and not 0.

{% call utils.code('python') %} >>> word[-1] # Gets the last character 'n' >>> word[-2] # Gets the second to last character 'o' >>> word[-6] 'P' {% endcall %}

If you try to use an index that's too large, you'll get an IndexError:

{% call utils.code('python') %} >>> word[42] # 'Python' only has 6 characters Traceback (most recent call last): File "", line 1, in IndexError: string index out of range >>> word[-8] Traceback (most recent call last): File "", line 1, in IndexError: string index out of range {% endcall %} {% endcall %} {# TODO: Modularize #} {% call section('quiz', 'Quiz') %}
