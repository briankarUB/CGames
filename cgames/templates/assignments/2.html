Lists

In python, we use lists to store multiple elements. In general, all elements in the lists must be of the same type. For syntax, we use brackets: [ and ] for lists. For example, take a look at this list declaration {% call utils.code('python') %} myList = ['python', 'java', 'c sharp', 'ruby', 'perl'] {% endcall %} This list, which I've decided to call myList, is a list of five strings. Notice that I don't have to write (and cannot write) {% call utils.code('python') %} list myList = ['python', 'java', 'c sharp', 'ruby', 'perl'] {% endcall %} As you can see, my list has 5 elements. I can call len on myList to get the size of the list I've created. In this case len(myList) == 5.

Now let's say I want to add 'kuda' to myList. To do this, I use the append() function, like this: {% call utils.code('python') %} myList.append('kuda') {% endcall %} and the result is: {% call utils.code('python') %} ['python', 'java', 'c sharp', 'ruby', 'perl', 'kuda'] {% endcall %}

But, let's say I want 'kuda' at the beginning of my list. To do this, append() won't help me. Instead, I'll have to use the insert() method. insert() takes two arguments: the index where you want the new element to go and the element you want to add. Watch! {% call utils.code('python') %} myList.insert(0, 'kuda') {% endcall %} This tells Python to add 'kuda' to myList at index 0. The result is instead: {% call utils.code('python') %} ['kuda', 'python', 'java', 'c sharp', 'ruby', 'perl'] {% endcall %} 'kuda' is at index 0, and all other elements have been moved to the right (i.e., the index of every other element has been increased by 1).

Notice that I don't have to tell Python to make myList bigger to make room to add 'kuda'. It knows to do this and takes care of it automatically.

If I want to remove an element, I use the function remove(). remove() takes one argument: the element that you want to remove of the list. So, if I want to remove 'java', I'd write: {% call utils.code('python') %} myList.remove('java') {% endcall %} Then, if I call print(myList), this would print to the console: {% call utils.code('python') %} ['kuda', 'C++', 'python', 'ruby', 'perl'] {% endcall %}

Another handy function is the sort() function. If I call myList.sort(), this will sort the elements of myList in alphabetical order. In similar fashion, reverse() can be used to reverse a list. Check it out: {% call utils.code('python') %} >>> myList.sort() >>> print(myList) ['c sharp', 'java', 'perl', 'python', 'ruby'] >>> myList.reverse() >>> print(myList) ['ruby', 'python', 'perl', 'java', 'c sharp'] {% endcall %}

If Statements

In Python, there are times when we only want certain code to execute only some of the time. To control when code runs, we can use if statements. Let's look at some code to see how these work! {% call utils.code('python') %} >>> myList = ['bills', 'dolphins', 'jets', 'patriots'] >>> if len(myList) == 4: ... print('This list has four elements.') ... else: ... print('Hello, world') This list has four elements. {% endcall %}

So, why does the third line run, but not the fifth line? Well, the first if statement evaluates to True, since myList does have a length of 4. Then, Python skips the else, since it only runs if the previous if statement evaluates to False.

Let's look at another example: {% call utils.code('python') %} >>> myList = ['bills', 'dolphins', 'jets', 'patriots'] >>> if myList[3] == 'bills': ... print('bills are last') ... elif myList[3] == 'dolphins': ... print('dolphins are last') ... elif myList[3] == 'jets': ... print('jets are last') ... else: ... print('patriots are last') patriots are last {% endcall %} Why? This time, all of the if statements evaulate to False. So, none of the code in the if statements are executed. Instead, the else statement is executed.

For Loops

Sometimes we want to iterate over, or read each element in a list. To accomplish that, we use a "for loop", like this: {% call utils.code('python') %} >>> myList = ['python', 'java', 'c sharp', 'ruby', 'perl'] >>> for s in myList: ... print(s) python java c sharp ruby perl {% endcall %} Here, the elements of myList get printed out, one at a time.
